# ADR 0001: Documentation Lives in GitHub
## Status
Accepted
## Context
По мере роста числа лендингов проекту требуется единое, проверяемое и доступное всем участникам место хранения операционных знаний. Если документация остаётся только в чатах, локальных заметках или устных договорённостях, команда теряет воспроизводимость: следующий исполнитель не видит актуальные правила, решения и логику изменений.
## Decision
Вся рабочая документация по фабрике лендингов должна жить в GitHub-репозитории проекта. Это правило распространяется на стандарты, playbooks, ADR, а также на проектные документы отдельных лендингов.
## Consequences
GitHub становится не только местом хранения кода, но и точкой входа в операционную память проекта. Любое существенное изменение процесса, структуры документации или проектной логики должно оформляться так, чтобы его можно было найти в репозитории и связать с историей изменений.
## Boundary
Это решение не означает, что в GitHub нужно хранить все временные материалы без разбора. Черновые заметки, промежуточные выгрузки и одноразовые рабочие файлы допустимы только как временный слой и должны либо удаляться, либо конденсироваться в системные документы.
