# ADR 0002: Project Docs Are Organized by Landing Slug
## Status
Accepted
## Context
Когда проект ведёт несколько лендингов параллельно, знания быстро начинают размазываться между root-уровнем, общими файлами и случайными заметками. В результате становится трудно понять, какие документы относятся к конкретному лендингу, а какие являются общими стандартами фабрики.
## Decision
Проектная документация каждого лендинга должна храниться в отдельной папке вида `docs/projects/<slug>/`. Идентификатор `slug` должен совпадать с именованием страницы, ассетной папки и других проектных сущностей.
## Consequences
У каждого лендинга появляется свой компактный пакет знаний, который легко передавать, ревьюить и расширять. Общие правила при этом остаются в слоях `docs/standards/`, `docs/playbooks/` и `docs/adr/`, а проектные документы не смешиваются с фабричными.
## Boundary
Если документ описывает правило, применимое ко всем лендингам, он не должен жить внутри `docs/projects/<slug>/`. Если же документ нужен только для одного лендинга, его нельзя оставлять в корне или в общем слое без явной причины.
